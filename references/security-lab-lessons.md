# Security Lab Lessons

在任务命中下面任一场景时读取本文件：

- 需要先圈定测试范围，再做协议录制或重放
- 需要把真实浏览器流量导出成可复查证据
- 需要在可复位实验环境里做带副作用的协议续跑
- 需要把评分、控制平面和被测实例分开

开始前先写清 4 项声明：

- scope：允许测试的域名、路径、账号、租户
- mode：本轮是 `observe_only`、`replay_only` 还是 `mutating_run`
- reset：如果产生副作用，如何复位
- evidence：本轮至少保留哪些原始工件

读完本文件后，至少应能产出一份简短决策：

- 当前 scope 是否足够清晰
- 当前 mode 是否只能停在 `observe_only`
- 若允许升级，升级前还缺哪一项复位或证据条件

## 1. 先定义上下文，再开始录制

ZAP 的 `Context` 机制强调：先定义 URL 范围，再附加认证、会话与用户信息。

- 上下文文档：<https://www.zaproxy.org/docs/desktop/start/features/contexts/>
- 自动化框架：<https://www.zaproxy.org/docs/automate/automation-framework/>

可迁移经验：

- 先写清测试域名、路径前缀、账号、租户和允许访问的入口，再开始抓流量。
- 把“范围定义”与“执行步骤”分开保存，避免测试跑着跑着越界。
- 对自动化执行保留显式作业清单和退出条件，不要只留下零散脚本。

## 2. 录制与重放要分开对待

mitmproxy 把保存、导出、客户端重放、服务端重放做成独立能力，而不是一个黑箱动作。

- 功能总览：<https://docs.mitmproxy.org/stable/overview/features/>

可迁移经验：

- 同时保留原始流量工件和归一化摘要，二者不要互相替代。
- 重放前先声明匹配依据：URL、方法、头部、cookie 还是请求体。
- 把“最小重放”限定为证明协议行为所需的最少请求集，不包含会创建、删除或触发次生状态变化的请求。
- 不把一次成功重放误判成“真实浏览器行为完全复现”；回放只能证明局部链路可再现。

## 3. 带副作用的测试必须优先选择可复位环境

Juice Shop 的 CTF 模式强调把题目状态、评分服务器和参与者实例拆开，并支持按实例隔离。

- CTF 模式文档：<https://pwning.owasp-juice.shop/companion-guide/latest/part4/ctf.html>

可迁移经验：

- 能用一次性租户、临时实例、独立账号时，不要直接在共享环境里做变更测试。
- 被测实例和观测/评分平面尽量分离，避免让证据系统本身污染链路。
- 如果实例共享、单用户，或复位能力未知，就把环境视为只允许观测和录制，不允许副作用续跑。
- 每次实验前先写明“如何复位”，否则默认只做观测和录制。
- 共享实例只要复位能力不明确，就把任务降级成证据采集，不做 best-effort 续跑。

## 4. 控制平面和被测平面分离

Juice Shop 与 CTFd 的组合说明：挑战定义、计分、用户管理可以独立于具体被测应用运行。

- CTFd 仓库：<https://github.com/CTFd/CTFd>
- Juice Shop CTF 集成文档：<https://pwning.owasp-juice.shop/companion-guide/latest/part4/ctf.html>

可迁移经验：

- 把“实验编排、证据记录、结果判定”视为控制平面。
- 把“真实协议续跑、资源创建、callback 接收”视为被测平面。
- 控制平面只读取和判定，不应该偷偷修改被测链路中的关键状态。

## 5. 证据工件至少保留四类

建议每次协议实验至少保存：

- 范围声明：域名、账号、租户、允许副作用范围
- 原始工件：抓包流、HAR、302 链、邮件原文或验证码摘要
- 归一化状态：`state`、`code_verifier`、cookie 摘要、continue URL 来源
- 运行结论：当前模式、断点、是否 `degraded_path=true`、是否命中提交屏障

如果只能保留最终日志，而没有原始流量或来源标签，该结论默认只算弱证据。

进入真实副作用续跑前，至少再确认一次：

- 当前实例是否独占
- 复位步骤是否已经实测
- 谁批准了这次变更
- 失败后如何停止并留痕

建议补一张最小复位确认表：

| 资源或副作用 | 复位动作 | 是否实测 | 责任人 | 失败止损 |
| --- | --- | --- | --- | --- |
| 例如：测试账号、验证码、临时工作区、注册资源 | 删除 / 回收 / 重建 / 清 cookie | 是 / 否 | 谁批准并执行 | 失败后立即停在哪一步 |
